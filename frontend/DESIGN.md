# 역사로 — 디자인 시스템 (DESIGN.md)

> 단일 진실 소스(Single Source of Truth). 새 UI를 만들거나 고칠 때 이 문서의 토큰·원칙·패턴을 따른다.
> 사람이 읽어도, AI에게 "DESIGN.md를 따라 이 섹션을 다듬어줘"라고 시켜도 일관되게 나오도록 작성한다.

---

## 1. 디자인 방향 (한 문장)

**조선 헤리티지의 따뜻함 × 토스/당근식 미니멀.**
한지·먹·단청의 정체성은 색과 폰트로만 남기고, 레이아웃·여백·인터랙션은 현대적이고 절제되게.

원칙:
- 장식보다 위계. 그라데이션·글로우·굵은 테두리 대신 여백·타이포·정렬로 정리한다.
- 그림자는 단층·은은하게. 떠 보이게 하지 말고 살짝 들어올린다.
- 색은 절제. 화면당 강조색(단청 적)은 1~2곳. 보라는 "상상" 구간에만.
- 모션은 거들 뿐. 진입 시 한 번 부드럽게, 무한 반복 애니메이션 금지.
- AI 티 금지. 본문에 이모지·볼드 남발 금지(평문).

## 2. 색 (Color tokens) — `:root`에 정의됨, 신규 색 추가 금지

| 역할 | 토큰 | 값 |
| --- | --- | --- |
| 배경 기본 | `--bg` | #FFFFFF |
| 배경 종이 | `--bg-subtle` | #F8F7F5 |
| 배경 진한 | `--bg-muted` | #EDEBE6 |
| 먹색(본문) | `--ink` | #1A1612 |
| 먹색 연함 | `--ink-soft` | #5C5549 |
| 먹색 흐림 | `--ink-faint` | #9A9088 |
| 단청 적(포인트) | `--red` / `--red-deep` | #C8453B / #A0352C |
| 상상 보라(상상 구간 전용) | `--imagine` / `--imagine-deep` | #6B5BD0 / #4E3FB8 |
| 테두리 | `--border` / `--border-muted` | #E2DDD7 / #EDEAE5 |

사용 규칙:
- **적(red)** = 행동(CTA)·브랜드·"사실(史)" 강조. **보라(imagine)** = "만약에/상상" 구간에만. 둘을 한 요소에 섞지 않는다.
- 텍스트 위계는 `--ink`(제목·본문) → `--ink-soft`(보조) → `--ink-faint`(메타/플레이스홀더) 3단으로만.

## 3. 타이포그래피

| 역할 | 폰트 | 쓰임 |
| --- | --- | --- |
| 디스플레이 | Black Han Sans | 큰 타이틀(히어로 h1), 아주 절제해서 |
| 제목·버튼 | Jua | 섹션 제목(h2/h3), 버튼 라벨 |
| 본문 | Gowun Dodum | 설명·문단·캡션 |

타입 스케일(권장, px):
`32 / 26 / 21 / 18 / 16(기본) / 14 / 13 / 12`
- 히어로 h1만 28~36, 나머지 섹션 제목은 21~26로. 같은 위계는 항상 같은 크기.
- 줄간격: 제목 1.25, 본문 1.6, 캡션 1.4. 한글은 `word-break: keep-all`.

## 4. 간격 (Spacing) — 4의 배수만

`4 · 8 · 12 · 16 · 20 · 24 · 32 · 40 · 56 · 80`
- 섹션 상하 패딩: 데스크탑 80, 모바일 56.
- 카드 내부 패딩: 24~32.
- 요소 간 기본 갭: 12~16.
- "깔끔함"의 8할은 여백. 채우려 하지 말고 비운다.

## 5. 모양·그림자·모션

- 라운드: 카드 `--radius`(16), 작은 요소 `--radius-sm`(10), pill 999px. (로그인 카드만 20)
- 그림자: `--shadow-xs`(살짝) / `--shadow`(카드) / `--shadow-lg`(떠야 할 때). 직접 rgba 그림자 새로 만들지 말 것.
- 테두리: 1px `--border`. 1.5px 이상 굵은 테두리 금지.
- 모션: 진입 fade+rise(translateY 8~16px, 0.4~0.55s, `cubic-bezier(.22,1,.36,1)`). hover는 색/배경만, 과한 scale·그림자 점프 금지. `prefers-reduced-motion` 항상 존중.

## 6. 컴포넌트 패턴

- **버튼(주)**: 솔리드 `--red`, 흰 글자, radius 12, padding 14×16, hover `--red-deep`. 그라데이션·광택 금지.
- **버튼(보조)**: 흰 배경 + 1px `--border` + `--shadow-xs`, hover 배경 `--bg-subtle`.
- **텍스트 링크**: 배경·테두리 없는 `<button>`(리셋 필수: `background:none;border:none;padding:0`), 색 `--red`, hover 밑줄.
- **카드**: 흰 배경, 1px `--border`(또는 무테+그림자), `--radius`, `--shadow`. 내부 패딩 24~32.
- **칩**: pill, 1px 테두리 또는 솔리드(이동/강조용은 솔리드 적).
- **입력**: 흰 배경, 1px `--border`, radius 12, focus 시 `--red` 테두리 + 3px 적 글로우 링.
- **섹션 헤더**: eyebrow(작은 대문자/적색) → 제목(Jua) → 보조설명(ink-soft). 가운데 또는 좌측 정렬 일관되게.

## 7. 안티패턴 (하지 말 것)

- 한 화면에 강조색 3곳 이상 / 적·보라 혼용
- 그라데이션 배경·버튼, 글로우 헤일로, 1.5px+ 굵은 테두리
- 무한 반복 애니메이션, 과한 hover scale
- 빽빽한 레이아웃(여백 부족), 같은 위계인데 제각각 크기
- 본문 이모지·볼드 남발

## 8. 메인 페이지 리디자인 계획 (섹션별, 우선순위)

현재 홈 구성: `Hero → HowItWorks → EventGrid → Timeline → Gallery → SillokPillars → Footer`

1. **글로벌 정리(P0)** — 섹션 패딩·제목 크기·강조색 사용을 위 토큰으로 통일. (구조 변경 없이 일관성만 끌어올림 → 체감 큼)
2. **Hero(P0)** — 첫인상. 카피·여백·CTA 위계 정리, 장식 절제. 세종 마스코트 톤 정돈.
3. **EventGrid(P1)** — 작품 카드 통일(썸네일·제목·메타·CTA 정렬), factCard 2줄 clamp, 그림자/라운드 일관.
4. **HowItWorks(P1)** — 3스텝 카드 여백·번호 위계 정리.
5. **Timeline / Gallery / SillokPillars(P2)** — 톤 맞추고 과한 장식 제거.
6. **Footer(P2)** — 간결 정리.

각 단계는 독립 배포 가능. P0(글로벌+Hero)부터 시작 권장.
